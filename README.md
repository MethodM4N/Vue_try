# vue_try

Данное приложение было создано в результате обучения на utube канале UlbiTV.

## Project setup

```
npm install
```

### Compiles and hot-reloads for development

```
npm run serve
```

### Compiles and minifies for production

```
npm run build
```

## Стэк технологий

```
- HTML5
- CSS3
```

```JS
- JavaScript ES2015
- Vue3
- Vuex
```

## Другие технологии/функционал

```
- Создание постов (v-for)
- Двусторонние связывания (v-model)
- Декомпозиция
- Обмен данными между компонентами (передача данных в родительский компонент при помощи $emit)
- Создание UI компонентов
- Глобальная регистрация компонентов (app.component (в main.js))
- Popup окно
- Axios (загрузка данных с https://jsonplaceholder.typicode.com/)
- Жизненные циклы компонентов (хуки Mounted(во время первого рендера страницы), Watch(когда какая та из моделей 
была изменена, ничего не возвращает), Computed(во время первого рендера страницы, функция которая должна всегда 
что-то возращать, запоминает результат вычислений, будет вызвана если изменится зависимость, 
которая используется в этой функции))
- Отрисовки по условию (индикатор загрузки данных)
- Анимации (transition-group)
- Поиск постов
- Пагинация (axios params)
- Динамическая пагинация бесконечная лента (intersection observer API, ref обращение к DOM элементу)
- Постраничная навигация (VUE-ROUTER, use(router) в main.js)
- Динамическая навигация (/posts/:id, @click="$router.push(`/posts/${post.id}`))
- Создание собственных директив (app.directive в main.js, сделали el.focus() на инпуты)
- Mixins (создали примесь данных модального окна компонента Popup)
- Vuex v4 (глобал стор, библиотека для работы с состоянием, use(store) в main.js, state, getters 
(computed свойства, кэшируемые вычисляемые значения), mutations (функции внутри которых мы меняем state), 
action(функции которые внутри себя используют мутации, в них нельзя изменять состояния), 
commit нужен для вызова мутация, dispatch нужен для вызова других экшенов)
- Composition API (с помощью него можно делать декомпозицию отдельных логических кусков внутри одного компонента 
(особенно полезно, когда наш компонент разросся на 1000 строк кода), добавляет больше гипкости в код, 
куски можно группировать в отдельные библиотеки, которые при этом не будут терять реактивность)
```
